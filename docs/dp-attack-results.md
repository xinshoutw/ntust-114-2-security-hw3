# Step 3 DP CNN Attack Results

本頁整理 DP 影像在 CNN 重識別攻擊下的結果。DP 影像放在 `data/dp/`（四種 DP-Pixelization × 兩種 DP-Blur 機制 × 7 個 ε，共 42 組），效用指標 MSE/SSIM 在 [`data/dp/metrics.csv`](../data/dp/metrics.csv)，敏感度推導見 [`Differentially_Private_Image_Obfuscation.pdf`](Differentially_Private_Image_Obfuscation.pdf)。

每一個 method × ε 都獨立訓練一個 CNN，沒有混合訓練。評估使用 Step 2 同一份 `data/splits/{split_train,split_val,split_test}.json`，best checkpoint 由 val accuracy 挑選、Top-1/Top-5 在 held-out test set 上量測（N=80）。

執行重現：

```bash
uv run --extra attack python scripts/train_evaluate_dp.py --device auto --output reports/dp_evaluation.csv
uv run --extra attack python scripts/plot_curves.py    # 重畫兩張 DP 曲線
```

## 機制摘要

| 機制 | 公式（per-pixel Laplace scale）| 嚴格 ε-DP? | 用途 |
|---|---|---|---|
| **DP-Pixelization (b)** | 對每個 b×b cell mean 加 `Laplace(0, 255/(b²·ε))` | 是（parallel composition；不同 cell 用不同 pixel）| 主要 DP 方法 |
| **LP-Blur (k=45)** | 對每個 output pixel 加 `Laplace(0, 255/(k²·ε))` | **否**（沒做 composition）| 對照組：「naive blur+noise」 |
| **DP-Blur-Split (k=45)** | 對每個 output pixel 加 `Laplace(0, 255·k²/ε)` | 是（sequential composition 給 k² 個受影響 output）| 對照組：嚴格 DP 的代價 |

`LP-Blur` 與 `DP-Blur-Split` 共用同一個 k=45 Gaussian blur，差別只在加噪 budget 怎麼配置。

## Analysis

- **DP-Pix 呈現清晰的 privacy-utility 曲線**：b=2 在 ε=0.1 把攻擊壓到 8.75%（接近隨機 2.5%），ε 升到 5 回爬到 88.75%。較大的 b 在低 ε 時保護較弱（sensitivity 255/b² 較小、所需 noise 較少），但 utility 較高。
- **LP-Blur 沒有 DP 保護**：noise σ 在 k=45 下只有 ~0.18，遠小於像素範圍。ε 不影響輸出、攻擊準確率穩定在 ~85%。作為「naive blur+noise」對照組。
- **DP-Blur-Split 嚴格 DP 但 utility 歸零**：scale = 255·k²/ε，影像被噪音淹沒（SSIM=0.003），CNN 攻擊 ≈ 隨機。Strict DP 需要的雜訊量直接抹掉所有可辨識訊號。

## Caveats

- N=80：1 筆預測 = 1.25pp。LP-Blur ε=0.5 = 0.5375 是 outlier（MPS non-determinism），同方法其他 6 個 ε 都在 0.84-0.89；LP-Blur 應視為 ~0.85 ± noise。
- DP-Blur-Split 的 255-per-output sensitivity 是寬鬆上界（緊上界約 0.18·255 ≈ 46）。本實作多加了一些雜訊以確保 ε-DP 嚴格成立。
